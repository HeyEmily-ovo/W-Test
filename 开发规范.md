# W-Test 测试开发规范

本文档定义了新增心理学测试的标准开发流程和代码规范。添加新测试时，请严格按照本规范执行，确保一致性。

---

## 1. 快速检查清单

新增测试需完成以下步骤：

- [ ] 在 `tests/<id>/` 下创建目录和文件（3 个文件：`index.html`、`style.css`、`script.js`）
- [ ] 在 `js/config.js` 的 `TEST_CONFIG` 数组中添加条目
- [ ] 实现 5 个页面：Landing → Test → Results → History → Encyclopedia
- [ ] 实现全局函数：`startTest`、`showResults`、`showHistory`、`showEncyclopedia`、`backToLanding`、`retakeTest`、`shareResult`、`clearHistory`
- [ ] 实现 `saveToHistory()` 和 `renderHistoryList()`
- [ ] 添加 `<link rel="icon">` 引用 `../../favicon.svg`
- [ ] 添加 `.wt-nav` 返回导航栏
- [ ] 响应式适配（断点：`680px`、`480px`）
- [ ] 所有文字和注释使用简体中文

---

## 2. 目录与文件结构

```
tests/<id>/
├── index.html       # 页面结构（5 个 section）
├── style.css        # 样式（含 :root 变量 + .wt-nav + 响应式）
└── script.js        # 逻辑（题目数据 + 计算 + 历史 + 图鉴）
```

> 使用**三文件分离**模式（参考 history/love/maturity/animal），不再使用 mbti 的单文件内联模式。

### 2.1 测试 ID 命名

- 使用**全小写英文短横线命名**（kebab-case），如 `enneagram`、`dark-triad`
- `id` 同时作为目录名、`localStorage` key 前缀、config 中的标识符
- 避免使用拼音，除非该测试名称已约定俗成

### 5.5 saveToHistory 标准实现

```javascript
function saveToHistory() {
    const entry = {
        id: Date.now(),
        date: new Date().toLocaleString('zh-CN', {
            year: 'numeric', month: '2-digit', day: '2-digit',
            hour: '2-digit', minute: '2-digit'
        }).replace(/\//g, '/'),
        resultType: currentResult.type,
        resultName: currentResult.name,
        resultEmoji: currentResult.emoji,
        scores: { ...scores }
    };
    
    try {
        const history = JSON.parse(localStorage.getItem(STORAGE_KEY) || '[]');
        history.unshift(entry);
        if (history.length > MAX_HISTORY) history.pop();
        localStorage.setItem(STORAGE_KEY, JSON.stringify(history));
    } catch (e) {
        console.warn('历史记录保存失败:', e);
    }
}
```

### 5.6 页面切换辅助函数

```javascript
function hideAll() {

function showSection(id) {
    hideAll();

    if (el) el.classList.add('active');
}
```

## 6. 题目数据结构

### 6.1 统一 Likert 量表题（推荐）

适用场景：态度/倾向类测试。选项文本对所有题目相同，分数递减。

```javascript
const QUESTIONS = [
    {
        id: 1,
        dim: 'dimension_key',     // 所属维度
        text: '题目文本',
        dimension: '维度中文名',   // 用于展示
        reversed: false            // false=正向计分, true=反向计分
    }
];

// 全局 Likert 选项（5 级）
const LIKERT_OPTIONS = [
    { label: '完全符合', value: 5 },
    { label: '比较符合', value: 4 },
    { label: '一般',     value: 3 },
    { label: '不太符合', value: 2 },
    { label: '完全不符', value: 1 }
];
```

### 6.2 情境选择题

适用场景：行为偏好类测试。每题的选项文本和分数各不相同。

```javascript
const QUESTIONS = [
    {
        id: 1,
        dim: 'dimension_key',
        type: 'situational',
        text: '题目文本',
        dimension: '维度中文名',
        options: ['选项A描述', '选项B描述', '选项C描述', '选项D描述'],
        scores: [4, 3, 2, 1]                // 对应选项的分数
    }
];
```

### 6.3 混合类型（Likert + 情境混合）

适用场景：需要灵活度的复杂测试。参考 `tests/maturity/script.js`。

```javascript
const QUESTIONS = [
    { id: 1, dim: 'a', type: 'likert',      text: '...', reversed: false },
    { id: 4, dim: 'b', type: 'situational', text: '...', options: [...], scores: [...] }
];
```

### 6.4 多维积分题

适用场景：每个选项可能给多个维度加分（参考 `tests/animal/script.js`）。

```javascript
const QUESTIONS = [
    { q: '题目文本', o: [
        { t: '选项A', p: { dimension1: 2, dimension2: 0 } },
        { t: '选项B', p: { dimension1: 1, dimension2: 1 } }
    ]}
];
```

## 7. 结果计算模式

根据测试类型选择合适的计算模式：

### 7.1 维度累分 + 条件判定（最常用）

```
各维度独立累分 → 按阈值判定结果类型
```
参考：`tests/love/script.js`

### 7.2 维度累分 + 最高分匹配

```
各维度独立累分 → 取最高分维度对应的结果
```
参考：`tests/animal/script.js`

### 7.3 维度百分比 + 距离匹配

```
归一化为百分比 → 与预定义 profiles 计算欧氏距离 → 取最近匹配
```
参考：`tests/history/script.js`

### 7.4 维度百分比 + 区间分级

```
归一化为百分比 → 按阈值判定等级区间
```
参考：`tests/maturity/script.js`

## 8. 图鉴实现模式

根据结果类型数量选择：

| 结果数量 | 推荐模式 | 参考 |

| ≤ 12 个 | **卡片网格 + 点击弹窗**（detail overlay） | animal |
| 12~20 个 | **卡片网格 + 翻转卡片**（click flip） | love、mbti |
| ≤ 8 个 | **静态详情列表** | maturity |

## 10. 代码风格约定

| 项目 | 约定 |
|---|---|
| 缩进 | 2 或 4 空格（与选择的模板测试保持一致） |
| 引号 | 单引号 `'` |
| 分号 | 使用分号 |
| 函数声明 | `function name() {}`（非箭头函数，便于 hoisting） |
| 变量声明 | `let` / `const`，禁用 `var` |
| 常量命名 | 全大写蛇形：`STORAGE_KEY`、`MAX_HISTORY`、`LIKERT_OPTIONS` |
| 函数命名 | 小驼峰：`startTest()`、`renderQuestion()` |
| 注释语言 | **简体中文** |
| 动画 | 统一使用 `fadeIn` keyframe（0.4s ease） |
| history 上限 | 30 条（`const MAX_HISTORY = 30`） |
| 错误处理 | localStorage 操作包裹 `try/catch` |
| emoji 风格 | 使用 emoji 作为图标和情绪表达，保持亲和力 |
| 按钮文案 | 统一："开始测试 →"、"📋 复制结果分享"、"🔄 重新测试"、"📋 历史记录"、"📖 [图鉴名称]" |

## 11. 常见问题

### Q: 如何快速开始一个新测试？
复制 `tests/love/` 或 `tests/maturity/` 目录作为模板，修改其中的题目数据、颜色变量和图鉴内容。

### Q: ECharts 图表是必需的吗？
不是。仅当结果需要雷达图或柱状图时引入。语法：在 `<head>` 中加 `<script src="https://cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js"></script>`。

### Q: 题目导航网格列数怎么定？
- 总共 ≤ 30 题：`grid-template-columns: repeat(6, 1fr)`（手机 5 列）
- 总共 > 30 题：`grid-template-columns: repeat(10, 1fr)`（手机 6 列）
